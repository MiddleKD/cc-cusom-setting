## Verification Strategy

> **ZERO HUMAN INTERVENTION** — ALL verification is agent-executed.
> Acceptance criteria requiring "user manually tests/confirms" are FORBIDDEN.

### Test Decision
- **Infrastructure exists**: [YES/NO]
- **Automated tests**: [TDD (RED-GREEN-REFACTOR) / Tests-after / None]
- **Framework**: [bun test / vitest / jest / pytest / none]

### QA Policy
Every task MUST include agent-executed QA scenarios.
Evidence saved to `.sisyphus/evidence/task-{N}-{scenario-slug}.{ext}`.

- **Frontend/UI**: Use Playwright — navigate, interact, assert DOM, screenshot
- **CLI**: Use Bash — run command, validate output, check exit code
- **API/Backend**: Use Bash (curl) — send requests, assert status + response fields
- **Library/Module**: Use Bash (node/bun REPL) — import, call, compare output

## Execution Strategy

### Parallel Execution Waves

```

├── Task 1: [description] [effort: quick/medium/deep]
├── Task 2: [description]
└── Task 3: [description]

Wave 2 (After Wave 1 — tasks with Wave 1 dependencies):
├── Task 4: [description] (depends: 1, 2)

Wave FINAL (After ALL implementation — parallel review):
├── F1: Plan Compliance Audit (oracle)
├── F2: Code Quality Review
├── F3: Real QA
└── F4: Scope Fidelity Check

Critical Path: Task 1 → Task 4 → F1-F4
```

| 2 | — | 4 |
| 3 | — | 5 |

| 5 | 3 | F1-F4 |

  - [Test cases to cover, if TDD]

  - [Pattern to avoid]

  - **Blocked By**: None (can start immediately) | Tasks N, M

  *Pattern References* (existing code to follow):
  - `src/services/auth.ts:45-78` — JWT creation and refresh token pattern

  *Type/API References* (contracts to implement against):
  - `src/types/user.ts:UserDTO` — response shape for user endpoints

  *Test References* (testing patterns):
  - `src/__tests__/auth.test.ts:describe("login")` — test structure and mocking

  **Acceptance Criteria**:

  > Agent-executable verification ONLY. No human action permitted.

  - [ ] `[command]` exits 0
  - [ ] `[command]` outputs `[expected value]`

  **QA Scenarios** (MANDATORY — task is INCOMPLETE without these):

  ```
  Scenario: [Happy path — what SHOULD work]
    Tool: [Playwright / Bash / curl]
    Preconditions: [exact setup state]
    Steps:
      1. [exact action — specific command/selector/endpoint]
      2. [next action with expected intermediate state]
      3. [assertion — exact expected value]
    Expected Result: [concrete, binary pass/fail]
    Failure Indicators: [what specifically would mean failure]
    Evidence: .sisyphus/evidence/task-1-happy-path.png

  Scenario: [Failure/edge case — what SHOULD fail gracefully]
    Tool: [Bash / curl]
    Preconditions: [invalid input / error state]
    Steps:
      1. [trigger the error condition]
      2. [assert error is handled correctly]
    Expected Result: [graceful failure with correct error message/code]
    Evidence: .sisyphus/evidence/task-1-error-case.txt
  ```

  > Specificity requirements — every scenario MUST use:
  > - **Selectors**: `.login-button`, not "the login button"
  > - **Data**: `"test@example.com"`, not `"[email]"`
  > - **Assertions**: exact values, not "verify it works"
  > - **Negative**: at least ONE failure/error scenario per task

## Final Verification Wave

> 4 review agents run in PARALLEL after ALL implementation tasks.
> ALL must APPROVE before work is complete.
> Present consolidated results. Rejection → fix → re-run → present again.

- [ ] F1. **Plan Compliance Audit** — `oracle`
  Read plan end-to-end. For each "Must Have": verify implementation exists. For each "Must NOT Have": search codebase for forbidden patterns (reject with file:line if found). Check evidence files in `.sisyphus/evidence/`. Compare deliverables against plan.
  Output: `Must Have [N/N] | Must NOT Have [N/N] | Tasks [N/N] | VERDICT: APPROVE/REJECT`

- [ ] F2. **Code Quality Review** — general-purpose
  Run `tsc --noEmit` + linter + tests. Review all changed files for: `as any`/`@ts-ignore`, empty catches, `console.log` in prod, commented-out code, unused imports. Check AI slop: excessive comments, over-abstraction, generic names (data/result/item/temp).
  Output: `Build [PASS/FAIL] | Lint [PASS/FAIL] | Tests [N pass/N fail] | VERDICT`

- [ ] F3. **Real Manual QA** — general-purpose
  Start from clean state. Execute EVERY QA scenario from EVERY task — exact steps, capture evidence. Test cross-task integration. Test edge cases: empty state, invalid input, rapid actions. Save to `.sisyphus/evidence/final-qa/`.
  Output: `Scenarios [N/N pass] | Integration [N/N] | Edge Cases [N tested] | VERDICT`

- [ ] F4. **Scope Fidelity Check** — general-purpose
  For each task: read "What to do", read actual diff. Verify 1:1 — everything in spec was built, nothing beyond spec was built. Check "Must NOT do" compliance. Detect cross-task contamination.
  Output: `Tasks [N/N compliant] | Contamination [CLEAN/N issues] | VERDICT`

## Plan Naming Convention

File: `.sisyphus/plans/{slug}.md`

Slug rules:
- Lowercase, hyphen-separated
- Descriptive but short: `add-auth`, `refactor-api-layer`, `payment-flow`
- No dates or version numbers in filename
